<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Env -> Process | API keys enter the process via env vars; must never reach logs or LLM context |
| Process -> Log sink | structlog JSON output goes to stdout/file; must be credential-free |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Information Disclosure | structlog event_dict containing api keys | mitigate | Task 1 — `_redact` processor scans known credential-shaped values + key names before JSONRenderer; covered by 10 redaction tests |
| T-01-02-02 | Information Disclosure | `repr(settings)` leaking secrets | mitigate | Task 2 — All secret fields use `pydantic.SecretStr`; covered by `test_config.py::test_repr_does_not_leak_secrets` |
| T-01-02-03 | Tampering | Missing required env var causing silent fallback | mitigate | Task 2 — Pydantic Settings raises `ValidationError` at construction; `gekko doctor` (Plan 01-01) surfaces the missing var with a clear error |
| T-01-02-04 | Repudiation | Log line forgery via injected newlines in user-supplied strings | accept | structlog's JSONRenderer escapes control chars; not a P1 concern beyond the renderer's defaults |
</threat_model>
