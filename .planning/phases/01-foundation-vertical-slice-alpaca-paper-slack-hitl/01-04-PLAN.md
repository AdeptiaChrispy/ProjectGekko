<objective>
Land the load-bearing append-only audit log with a SHA-256 hash chain enforced in application code (per D-16). Three modules: `canonical.py` (one-shot lock-in of the canonical JSON shape so the chain survives schema-additive migrations), `log.py` (the single `append_event()` writer every other module uses), `verify.py` (the `walk_chain()` integrity checker invoked by `gekko audit verify`). Every `decision`, `proposal`, `approval`, `rejection`, `order_submitted`, `fill`, `kill_switch`, `cap_rejection`, `error` event written in Phase 1 — and every future phase — flows through this module.

Purpose: AUDT-01 ("every decision, every order, every fill, every cap rejection, every kill-switch event is recorded as an append-only audit entry") and AUDT-02 ("entries include actor, action, inputs, outputs, rationale, row hash chained to previous entry") are the immutable foundation of the "structured rationale-as-artifact" differentiator (D-12, D-15). The canonical JSON shape is a one-shot architectural decision per Claude's Discretion in CONTEXT.md — if the planner picks the wrong shape, every event written before correction is lost. RESEARCH §Pattern 3 explicitly flags this as a "lock-it-down-now" decision.

Output: `await append_event(session, user_id=..., strategy_id=..., event_type="decision", payload={...})` writes a single `Event` row with `prev_hash` from the previous max-id row and `row_hash` computed deterministically; `await walk_chain(session, user_id)` returns `[]` for an intact chain and `[id_of_broken_row]` if anyone tampers with `payload_json`, `event_type`, `ts`, or `user_id` of an existing row.
</objective>

    ## canonical.py behaviors (tests/unit/test_audit_canonical.py)

    1. **Sorted keys, no whitespace, ASCII-safe.** `canonical_json({"b": 1, "a": 2})` returns `'{"a":2,"b":1}'` (sorted keys, no spaces, no newline).

    5. **Nested dicts/lists preserved.** `canonical_json({"a": {"b": [1,2,3]}})` returns `'{"a":{"b":[1,2,3]}}'` (no whitespace anywhere, sorted at every dict level).

    7. **Non-ASCII handled.** `canonical_json({"text": "café"})` returns ASCII-only output (`é` escape) so byte-stable hashing works regardless of locale.

    **`src/gekko/audit/log.py`**

      import asyncio
      import hashlib
      from datetime import datetime, timezone
      from sqlalchemy import select
      from sqlalchemy.ext.asyncio import AsyncSession
      from gekko.audit.canonical import canonical_json, GENESIS_PREV_HASH
      from gekko.db.models import Event

      _append_lock = asyncio.Lock()

      async def append_event(
          session: AsyncSession,
          *,
          user_id: str,
          strategy_id: str | None,
          event_type: str,
          payload: dict,
      ) -> Event:
          """Single-source-of-truth append for the audit chain.

          - user_id-scoped chain (per D-21)
          - prev_hash = previous max-id row's row_hash for the user (or GENESIS_PREV_HASH if first)
          - payload_json column stores the FULL canonical subset string (so verify is one-liner)
          - row_hash = sha256(prev_hash.encode("ascii") + payload_json.encode("utf-8"))

          Concurrent callers within the same process are serialized by `_append_lock`.
          Cross-process concurrency is NOT a P1 concern (D-18 single-process modular monolith).
          """
          async with _append_lock:
              last_q = (
                  select(Event.row_hash)
                  .where(Event.user_id == user_id)
                  .order_by(Event.id.desc())
                  .limit(1)
              )
              last = (await session.execute(last_q)).scalar_one_or_none()
              prev_hash = last or GENESIS_PREV_HASH
              ts = datetime.now(timezone.utc).isoformat()
              canonical = canonical_json({
                  "event_type": event_type,
                  "payload": payload,
                  "ts": ts,
                  "user_id": user_id,
              })
              row_hash = hashlib.sha256(
                  prev_hash.encode("ascii") + canonical.encode("utf-8")
              ).hexdigest()
              row = Event(
                  ts=ts,
                  user_id=user_id,
                  strategy_id=strategy_id,
                  event_type=event_type,
                  payload_json=canonical,
                  prev_hash=prev_hash,
                  row_hash=row_hash,
              )
              session.add(row)
              await session.flush()
              return row

    **`src/gekko/audit/verify.py`**

      from sqlalchemy import select
      from sqlalchemy.ext.asyncio import AsyncSession
      from gekko.audit.canonical import GENESIS_PREV_HASH
      from gekko.db.models import Event
      import hashlib

      async def walk_chain(session: AsyncSession, user_id: str) -> list[int]:
          """Return ids of rows whose hash chain is broken; [] if intact.

          Per-user scope. payload_json IS the canonical-subset string (set by append_event),
          so re-hashing is a one-liner without re-parsing.
          """
          q = select(Event).where(Event.user_id == user_id).order_by(Event.id.asc())
          rows = (await session.execute(q)).scalars().all()
          breaks: list[int] = []
          expected_prev = GENESIS_PREV_HASH
          for r in rows:
              recomputed = hashlib.sha256(
                  expected_prev.encode("ascii") + r.payload_json.encode("utf-8")
              ).hexdigest()
              if r.row_hash != recomputed or r.prev_hash != expected_prev:
                  breaks.append(r.id)
              expected_prev = r.row_hash
          return breaks
