<objective>
Land HITL-05 end-to-end: a per-user + per-strategy quiet-hours predicate that pauses the agent loop entirely during the window (per D-46) and a DM-routing wrapper that suppresses routine-category DMs while letting safety-critical bypass categories (kill, errors, first-live fills) always fire (per D-48). The predicate is IANA-timezone-aware via stdlib `zoneinfo` (per D-49); strategy overrides win when set (per D-47).

Purpose: HITL-05 is the safety floor against 2am pings. Without this plan, the v1 swing-horizon agent fires Researcher + Decision cycles at any hour APScheduler triggers them, and routine fill DMs land regardless of the operator's sleep schedule. The DM bypass split (D-48) is the trade-off: safety surfaces (kill, errors, first-live fills) must always reach the operator; routine fill confirmations + daily P&L summary can wait until window-open or fire at their fixed schedule.

Output:
- `src/gekko/approval/quiet_hours.py` NEW module with `_resolve_quiet_hours(user_id, now, strategy_name=None) -> bool` async predicate.
- `src/gekko/execution/executor.py` EXTENDED: `_send_slack_dm_respecting_quiet_hours(user_id, text, *, category)` wrapper next to the existing `_send_slack_dm`; routine fill DM call site in `on_fill_event` rerouted through it; bypass-category DMs (MarketClosed line 454, BrokerOrderError line 654, cap_rejection branches) continue calling `_send_slack_dm` directly (also documented by Pitfall-9 AST gate so future contributors do not accidentally route them through the wrapper).
- `src/gekko/agent/runtime.py` EXTENDED: `trigger_strategy_run` consults `_resolve_quiet_hours(user_id, now, strategy_name)` at the top of the scheduled-source branch; when True and `source="schedule"`, the cycle returns immediately. When `source="manual"` (the existing `gekko run <strategy>` CLI path), the gate is bypassed per D-46 manual-override semantics.
- AST gate `tests/unit/test_quiet_hours_dm_gate.py` enforces that every `_send_slack_dm` call site is classified (Pitfall 9 from RESEARCH).
- Unit tests cover overnight-wrap, DST spring-forward, DST fall-back, strategy-override-wins, bypass-categories, routine-suppressed.
- Integration test `tests/integration/test_scheduler_quiet_hours.py` verifies APScheduler `trigger_strategy_run` is skipped when in window.
</objective>
