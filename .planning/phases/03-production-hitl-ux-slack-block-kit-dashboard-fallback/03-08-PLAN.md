<objective>
Close BLOCKER CR-01: eight safety-critical dashboard routes (POST /live-confirm, GET /live-confirm, POST /kill, POST /unkill, GET /kill/state, POST /strategies/{name}/promote-to-live, POST /trigger/{name}, and all /strategies* CRUD) lack require_session. Any local process can toggle the kill switch, promote a strategy to live, or confirm a real-money trade without credentials — defeating the HITL-06 dual-channel guarantee.

This plan implements fail-closed router-level auth (Depends(require_session) on the APIRouter constructor) with explicit public exemptions for /login, /healthz, and the Slack events bridge. Every previously-ungated route will derive user_id from the authenticated session, not from settings.gekko_user_id.

Purpose: The dual-channel HITL-06 gate is only meaningful if the second channel (dashboard) requires distinct authentication. Without auth the second channel is trivially bypassable by any co-located process.

Output: Updated routes.py with router-level auth dependency; new regression test file asserting unauthenticated → 302 on each safety-critical route.
</objective>

  Step 1 — Change the router constructor at line 49 to declare the default dependency:
    `router = APIRouter(dependencies=[Depends(require_session)])`

  Step 2 — Add explicit public exemptions. FastAPI allows per-route `dependencies=[]` to override the router default. Annotate the public routes with `dependencies=[]`:
  - `@router.get("/login", dependencies=[])` on login_get
  - `@router.post("/login", dependencies=[])` on login_post
  - `@router.get("/healthz", dependencies=[])` on healthz

For each gated route, add `user_id: str = Depends(require_session)` as a parameter (this is a second declaration that shadows the router-level one and provides the value).
