Close the HITL-02 partial gap (WR-08): _extract_retry_num in slack_handler.py reads body["headers"]["x-slack-retry-num"] which does not exist in Socket Mode (the production transport). The retry gate always returns 0 and is dead code in production. Purpose: Code that documents a guarantee it does not provide is worse than no code. Removing the gate makes the dedup contract clear: claim_action UNIQUE INSERT is the only idempotency layer, and it is both necessary and sufficient.
