## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /spend | Auth-gated route; unauthenticated requests redirected to /login |
| browser → /settings POST | daily_cost_ceiling_usd is user-submitted input; validated as Decimal before storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Elevation of Privilege | GET /spend data scope | mitigate | Route uses require_session → user_id; all DB queries filter by user_id (D-21 multi-tenant invariant); one user cannot view another's spend data |
| T-04-15 | Tampering | daily_cost_ceiling_usd form input | mitigate | Validated as Decimal before storage; Decimal("0") or negative rejected; stored as normalized str(Decimal) to prevent "00" or "1e2" style storage |
| T-04-16 | Denial of Service | User setting ceiling to $0.01 | accept | No minimum ceiling is technically required; the guard enforces whatever ceiling is set; this is the user's intentional choice (raise the ceiling to resume) |
| T-04-17 | Information Disclosure | spend.html.j2 template | mitigate | No external script src; extends base.html.j2 with existing CSP + vendored HTMX (SRI lint gate from Plan 01-09 covers this); spend data is scoped to authenticated user only |
| T-04-SC | Tampering | slopcheck | accept | No new packages installed |
</threat_model>
